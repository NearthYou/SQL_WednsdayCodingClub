SQL Processor Project Flow

하나의 SQL 파일이
파싱, 실행, 검증, 반영까지 이어지는 흐름

사용자가 SQL 스크립트를 입력하면 parser가 문장을 분석하고, executor가 staging 환경에서 안전하게 실행한 뒤, storage가 CSV와 schema 규칙에 맞춰 결과를 저장하거나 조회합니다.

핵심 축

  • Manual scanner 기반 SQL 파싱
  • Statement / SqlScript AST 구성
  • CSV + schema.csv 저장소 모델
  • Staging, commit, rollback 실행 보장
01 입력

SQL 파일 로드

main.c가 SQL 파일 전체를 읽고 실행할 데이터 디렉터리를 결정합니다.

queries/*.sql data/
02 파싱

구문 해석

parser.c가 키워드, 식별자, 값, 세미콜론 경계를 직접 스캔해 SQL 문장을 AST로 만듭니다.

INSERT SELECT multi-statement
03 AST

Statement / SqlScript

파싱 결과는 타입, 대상 테이블, 컬럼 목록, 리터럴 값을 담는 구조체로 정리됩니다.

schema.table values projection
04 실행기

Staging 실행

execute.c가 INSERT가 포함된 스크립트를 임시 staging 디렉터리에서 먼저 실행합니다.

buffered output safe commit
05 저장소

CSV / Schema 처리

storage.c가 schema 검증, 컬럼 재정렬, CSV append, projection SELECT를 담당합니다.

users.csv users.schema.csv type check
06 결과

출력과 반영

모든 문장이 성공하면 결과를 출력하고 CSV를 commit합니다. 중간 실패가 나면 rollback으로 원본을 유지합니다.

SELECT result commit rollback
Flow Rule

INSERT는 staging에 먼저 기록되고, 같은 스크립트 안의 SELECT는 그 staged 결과를 읽습니다. 전체가 성공했을 때만 원본 CSV가 갱신됩니다.
